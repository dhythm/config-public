## Context

- use `gh` command to enable to access the private repo
- `$ARGUMENTS` にユーザーから渡された「目的・背景の概要」が入る（空の場合もある）

## Instructions

対象の GitHub リポジトリに、**インテント（目的）のみを記載したイシュー**を作成します。
このイシューには「仕様」や「実装方法」は書きません。「なぜこれをやる必要があるのか」「どのような価値・効果を期待するのか」といったインテント（意図・背景・目的）のみを記載します。
仕様は実装着手時に別途検討するため、イシューでは判断の拠り所となる「やる理由」を残すことを目的とします。

### 構成情報

イシューは以下の情報で構成します。
内容によって項目を追加・削除しても構いません。ただし「仕様」「実装手順」「タスク分解」などは含めないでください。

- 背景 / 現状
- 目的（インテント）
- なぜ今やるのか
- 期待する効果・価値
- 関連情報（任意）

### 要件

- 見出しは h2 以降を使ってください
- 日本語で記載してください
- 仕様・実装方法・タスク分解は記載しないでください（インテントのみ）

### 引数のハンドリング

- `$ARGUMENTS` が**空でない場合**: その内容を「目的・背景の初期情報」として扱い、不足点を補うための質問を行う
- `$ARGUMENTS` が**空の場合**: まずユーザーに「どのようなインテントのイシューを作成したいか」を尋ねるところから開始する

### 作業詳細

1. `$ARGUMENTS` の内容を確認する
   - 内容がある場合は、それを目的・背景の初期情報として受け取る
   - 空の場合は、作成したいイシューのインテントについてユーザーに尋ねる
2. インテントを明確化するために「不足点・不明点」があればユーザーに質問し、回答を得る
   - 例:「なぜ今やる必要があるのか」「期待する効果は何か」「現状どのような課題があるのか」
3. 「不足点・不明点」がなくなるまでこの質問を繰り返す
4. 収集した情報を元に、「イシューの内容」（インテントのみ）を作成する
5. ユーザーに対して「イシューの内容」を提示し、問題ないかを尋ねる
6. ユーザーが「イシューの内容」を確認し、問題があればフィードバックする
7. ユーザーが承認するまで「イシューの内容」に対して修正を繰り返す
8. ユーザーが承認したら、GitHub にイシューを作成する
